# Format Conversion Step

Converts input documents from one format to another using extracted filter events. Supports output to PO, TMX, tab-delimited, Pensieve TM, parallel corpus files, and Word table (RTF) formats. The generated documents are not meant to be merged back into their original format — use the Raw Document to Filter Events / Filter Events to Raw Document step pair for round-trip scenarios.

Requires both source and target language to be set in the pipeline context.

## Parameters

#### Output Format

Select the output file format. Available formats:

- **PO File** — Gettext PO format, UTF-8 encoded
- **TMX Document** — Translation Memory eXchange format
- **Tab-Delimited Table** — UTF-8 tab-delimited text; source in first column, target in second; literal tabs escaped as `	`
- **Pensieve TM** — Pensieve TM database (output path must be a directory)
- **Parallel Corpus Files** — Two plain text files (inline codes stripped), one entry per line, locale code appended to filename. Used by systems like Moses MT
- **Word Table** — RTF table with generic inline codes; one entry per row. For TTX input, match percentage appears in the Type column

#### Target Content

Controls what content is placed in the target side of the output. The integer value selects between different target content strategies.

#### Create a Single Output Document

Generates one combined output document for all input documents. When disabled, each input document produces its own separate output file.

> **Note:** When enabled, the **Output path** field becomes active and must be specified.

#### Output Path

Full path of the single output document to generate. For a **Pensieve TM**, this must be a directory rather than a file path. The variable `${rootDir}` can be used in the path. Only active when **Create a single output document** is enabled.

Examples: `${rootDir}/output/combined.tmx`, `${rootDir}/pensieve-db/`

#### Output Paths with Format Extension

Output files use the same path as the input file with an additional format extension appended (e.g., `.po`, `.tmx`). When disabled, the output paths are determined by the calling application. Only available when **Create a single output document** is disabled.

#### Output Generic Inline Codes

Replaces original inline codes with generic placeholders in the form `<N>`, `</N>`, and `<N/>` where `N` is the code's numeric identifier. When disabled, inline codes are output in their original format (except for references).

> **Note:** If the output format is **TMX**, this option is ignored — inline codes always use TMX format regardless of this setting.

#### Do Not Output Entries Without Text

Suppresses output of entries that have no actual text content — entries that are empty or contain only inline codes with no surrounding text.

#### Output Only Approved Entries

Restricts output to entries where the target `approved` property is set to `yes`. For example, "fuzzy" entries in a PO file have this property set to `no`.

> **Note:** Entries that lack the `approved` property entirely are treated as **non-approved** and will be excluded.

#### Overwrite if Source is the Same (Pensieve TM)

When importing into a Pensieve TM, if an entry has the same source text as one or more existing TM entries, all matching entries are replaced by the new one. Leave disabled to allow multiple different translations for the same source text. Only applicable to Pensieve TM output.

## Limitations

None known.

## Notes

- Documents generated by this step are **not** round-trippable. For round-trip conversion, use the **Raw Document to Filter Events** step followed by the **Filter Events to Raw Document** step.
- For Pensieve TM output, the output path must point to a directory, not a file.

## Examples

**Single TMX output:**
```yaml
singleOutput: true
outputFormat: tmx
outputPath: "${rootDir}/output/combined.tmx"
skipEntriesWithoutText: true
```

**PO per input file (approved only):**
```yaml
singleOutput: false
autoExtensions: true
outputFormat: po
approvedEntriesOnly: true
```

**Parallel corpus for MT training:**
```yaml
singleOutput: false
autoExtensions: true
outputFormat: corpus
skipEntriesWithoutText: true
```
