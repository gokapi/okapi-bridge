# Vignette Filter

The Vignette Filter processes export/import XML documents used with the **Vignette Content Management System**. It handles files generated by the `vgnexport` tool of Vignette. Input and output encodings are forced to **UTF-8**. The filter extracts entries based on the `name` attribute of `<attribute>` elements and supports both bilingual (source+target matching via source ID) and monolingual modes.

## Parameters

### Attribute Elements to Extract

Comma-separated list of `name` attribute values identifying which `<attribute>` elements to extract from the Vignette export XML.

For example, if content is stored in `MYVCM-BODY`:
```xml
<attribute name="MYVCM-BODY">
  <valueCLOB>data</valueCLOB>
</attribute>
```
then include `MYVCM-BODY` in this list.

### Filter Configurations

Comma-separated list of filter configurations to apply to each extracted part. Must contain the **same number of entries** as the element names list, in the **same order**.

Use `default` to indicate no specific sub-filter for a given entry.

> **Note:** The number of entries must match the number of element names exactly.

### Monolingual Mode

Extracts **all** `importContentInstance` elements regardless of source/target pairing. When enabled, the source ID and locale ID parameters are ignored.

Use this for documents that contain only one language rather than source+target pairs.

### Source ID Element Name

The `name` attribute value of the `<attribute>` element that holds the **source ID** linking source and target `importContentInstance` blocks together.

Both the source and target blocks must share the same source ID value for the filter to pair them correctly.

> **Warning:** Entries with only a source `importContentInstance` (no matching target) are not extracted but generate a warning. Entries with no source `importContentInstance` at all are also skipped with a warning.

### Locale ID Element Name

The `name` attribute value of the `<attribute>` element that contains the **locale identifier** for a given content block.

## Limitations

- Very large documents may cause memory problems.
- Both source and target `importContentInstance` elements must be present for extraction in bilingual mode â€” missing either side produces a warning and skips the entry.

## Notes

- Input and output encodings are **forced to UTF-8** regardless of any encoding settings.
- Entries to extract are identified by the `name` attribute of `<attribute>` elements in the Vignette XML export.
- Source/target pairing is done via a shared source ID value across `importContentInstance` blocks.
- Monolingual mode bypasses the source/target pairing constraints entirely.

## Examples

### Vignette export XML structure

The filter processes XML with extractable `<attribute>` elements inside `importContentInstance` blocks:

```xml
<attribute name="MYVCM-BODY">
  <valueCLOB>data</valueCLOB>
</attribute>
```

### Extracting multiple attributes with sub-filters

Configure the filter to extract two attributes, applying an HTML filter to the body and no filter to the title:

```
Elements to extract: MYVCM-BODY,MYVCM-TITLE
Filter configurations: okf_html,default
```
