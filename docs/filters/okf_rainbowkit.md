# Rainbow Translation Kit Filter

The Rainbow Translation Kit Filter implements the `IFilter` interface for reading packages generated by the Rainbow Translation Kit Creation Step. It processes translation kit packages by reading their `manifest.rkm` file. This is a **read-only** filter — it cannot re-write translation kits, only read them for input processing.

## Parameters

### Open Manifest Before Processing

Opens the `manifest.rkm` file before processing begins, allowing you to inspect or modify it. This is useful for selectively merging back only a subset of files listed in the manifest rather than processing the entire package.

See the [Rainbow Translation Kit Manifest](Rainbow_Translation_Kit_Manifest) page for details on the manifest format and how to edit it.

> **Note:** The manifest file (`manifest.rkm`) is the entry point for the entire translation kit package — it must be used as the input file to process the package.

## Limitations

- This is a **read-only** filter. It does not generate output corresponding to the input — it must be used for input only.
- Cannot re-write or regenerate translation kit packages.

## Notes

- Rainbow translation kits are directory-based packages with a specific layout. The `manifest.rkm` file in the directory root serves as the entry point for processing the entire package.
- Different types of translation kit packages exist, each with its own directory structure and content.

## Examples

### Processing a translation kit

To process a Rainbow translation kit, point the filter at the `manifest.rkm` file inside the kit directory — not at individual files within the package.

```
myproject-tkit/
├── manifest.rkm      ← use this as input
├── original/
│   └── ...
└── work/
    └── ...
```
